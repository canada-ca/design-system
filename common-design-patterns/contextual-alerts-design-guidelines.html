Contextual alerts: Design guidelines

Alerts are notice boxes that bring users' attention to specific content that makes them aware of a result of a user action, a clarification or exception to surrounding content or to warn the user of a consequence of an action or inaction in their task.

Design specifications

  • 1. Icon

    Each alert type uses the following icons:

    • Danger alerts use an exclamation point icon in a circular border\e101
    • Warning alerts use an exclamation point icon in a triangular border \e107
    • Info alerts use an information icon \e086
    • Success alerts use a checkmark icon \e084
  • 2. Border highlight

    Alerts have border highlights with a thickness of 6px using the following colours:

    • Danger alerts use red #d3080c
    • Warning alerts use red #f90
    • Info alerts use red #269abc
    • Success alerts use red #278400
  • 3. Heading

    Alert heading text colour matches the border highlight colour

  • 4. Description

    Content area of an alert should inherit fonts and styles of the page

Usage

Do

Place an alert above the page heading if it applies to more than one page.

Do

Place an alert underneath a page heading if it applies to the whole page.

Do

Place an alert within a section of a page where it is relevant.

Do

If an alert advises of an issue or risk, it must provide a next step or suggested course of action to pursue.

Don't

Don't use an alert for content that will only be relevant to a small percentage of visitors to the page.

Do

Detect a user's situation to make an alert appear only to those it affects.

Don't

Don't shout (use caps) or use italics to call attention.

Don't

Alert messages should be polite and use plain language (~6-8 grade reading level).

Don't

Don't use an alert for form validation. Use for validation instead.

Consider labels or pills for styling of text with emphasis colours.

Do not use contextual alerts:

  • to provide information about a normal step in a process or task
  • to simply add emphasis to content
  • in cases where a warning is neither high risk nor frequently experienced
  • to create labels or pills. For these components, see:
    • label pattern
    • pill pattern

How to use

Alerts should be written for a grade 6-8 reading level.

Place the alert in the appropriate context:

Applies to the entire site
Place it at the top of the page above the main heading
Applies to the page
Place it underneath the page heading
Applies to a subsection of the page
Place it within that subsection, generally underneath the subsection heading or between 2 paragraphs

You can either use the beta or stable of version of this component. The beta version will eventually replace the stable one.

Use the beta contextual alerts component

Note: You will need to use custom CSS to implement these beta patterns.

Danger alert

When to use: Use the danger alert to draw attention to a situation that could place the user in danger, such as a do not travel warning. You can also use it to alert the user of a technical issue that could cause them to be unable to complete their task, such as an invalid submission of data in a form.

If pregnant or considering pregnancy - both partners

Avoid travel to Costa Rica if you are pregnant or planning to conceive a child in the next 3 months.

Code
HTML 
<section class="alert alert-danger">
 <h3>If pregnant or considering pregnancy - both partners</h3>
 <p>Avoid travel to Costa Rica if you are pregnant or planning to conceive a child in the next 3 months.</p>
</section>

Warning alert

When to use: Use the warning alert to draw attention to a possible consequence of an action or inaction, such as legal penalty that could apply.

Changes to impaired driving and cannabis-related crime laws

New penalties for impaired driving and cannabis-related crimes may affect your immigration status in Canada. If you commit one of these crimes, you may not be able to come to or stay in Canada.

<section class="alert alert-warning">

If you commit one of these crimes, you may not be able to come to or stay in Canada.</p>
</section>
