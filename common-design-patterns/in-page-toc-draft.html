In-page table of contents: Canada.ca design system

  • Last updated: 2021-01-15

The in-page table of contents pattern is a list of links that lead to subsections of the same page.

On this page

  • When to use
  • What to avoid
  • How to implement
  • Latest changes
  • Discussion

When to use

Use the in-page table of contents pattern to:

  • provide an outline for the content of a single page that has multiple subsections
  • provide an alternative method to jump to specific content on the page, without using the scroll bar

Use this pattern when:

  • excessive scrolling is required to view all of the content on the page
  • the page is divided into subsections, each with its own headings

What to avoid

  • Do not use this pattern when the document exists across several web pages
  • Only use one level - don't use sub-bullets

How to implement

  1. Unordered list
  2. Ordered list

1. Unordered list

When to use: Use an unordered list to give links to sections of a web page. Use "On this page" as a label.

What to avoid: Don't add sub-bullets for sub-sections.

On this page

  • Section 1
  • Section 2
  • Section 3
  • Section 4
Code 
<h2 class="h3">On this page</h2>
<ul>
   <li><a href="1">Section 1</a></li>
   <li><a href="2">Section 2</a></li>
   <li><a href="3">Section 3</a></li>
   <li><a href="4">Section 4</a></li>
</ul>

2. Ordered list

When to use: Use an ordered list for a table of contents of a document that has numbered headings. Use "Table of contents" as a haeading.

What to avoid: Don't add sub-bullet for sub-sections.

Table of contents

  1. Section 1
  2. Section 2
  3. Section 3
  4. Section 4
Code 
<h2 class="h3">Table of contents</h2>
<ol>
   <li><a href="1">Section 1</a></li>
   <li><a href="2">Section 2</a></li>
   <li><a href="3">Section 3</a></li>
   <li><a href="4">Section 4</a></li>
</ol>
