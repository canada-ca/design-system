Buttons: Design

  • Version 1.0
  • Last updated: [Date]

Buttons are clickable objects intended to encourage a person to take action.

Design Code Research

On this page

  • Design variants
  • When to use
  • Content guidelines
  • Accessibility guidelines

Design variants

Get code

1. Primary button

Use primary buttons for actions someone is likely to take on that page, such as the next or submit buttons in a form.

Get code

2. Secondary button

Use secondary buttons for alternative, less important or less likely to be used options, such as the 'previous' button in a form.

Get code

3. Link button

Use link buttons for infrequently used options, such as for restarting a wizard or calculator.

Get code

4. Call to action button

Use call to action buttons to start an application, survey, set of interactive questons or another experience when it is top task on that page.

Get code

5. Danger button

Use danger buttons to bring attention to an action that might have consequences, such as deleting a form or leaving an authenticated application.

When to use

Use the button pattern to execute actions or commands that will change the system state at the front or back end, for example “Add comment,” “Submit,” “Send.”

Use buttons when:

  • helping a person into or through a transaction such as “Sign in”
  • invoking functionality, for example “Share this page”
  • providing a persistent set of UI controls across a series of pages, for example “previous”, “next”, or “back to table of contents”

Buttons should

  • help a person into or through a transaction such as "Sign in"
  • invoking functionality, for example "Share this page"
  • provide a persistent set of UI controls across a series of pages, for example "previous", "next", or "back to table of contents"

Do not use buttons

  • when an ordinary text link will do the job just as well, like when providing a single, simple link to another page

Content guidelines

Use a label that clearly communicates the action that will be performed when the user clicks on it.

Accessibility guidelines

Mouse

Users can trigger a button by clicking anywhere within the button container.

Keyboard

Users can trigger a button by pressing Enter or Space while the button has focus.

Screen readers

VoiceOver: Users can trigger a button by pressing Enter or Space while the button has focus.

JAWS: Users can trigger a button by pressing Enter or Space while the button has focus.

NVDA: Users can trigger a button by pressing Enter or Space while the button has focus.
