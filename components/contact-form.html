Contact form: Design guidelines

Contact forms allow people to send a message to an owner of a website.

  • Version: 1.0
  • Updated: Feb 13, 2020
Design guidelines Demo and code Rationale

Design variants

1. Feedback form

Feedback forms lets visitors to send feedback to the website owner, usually to report a problem they experienced while trying to complete their task. They don't provide support for sending a response.

See the demo and code

2. Request form

Request forms let visitors request a response from the owner of the site by collecting contact details.

See the demo and code

3. Hybrid form

The hybrid variant lets visitors give feedback to the owner of the site, with the option of requesting a response.

See the demo and code

Usage

Do

Provide links to other contact channels that may be more convenient than the online form.

Do

When applicable, include a statement on the collection of personal information, after the form, either as a link to a separate page or in an expand-collapse.

Don't

Don't break the form fields across multiple columns. Keep in mind that it's easy for people using screen magnifiers to miss form fields that don't align to the left.

Do

Provide a confirmation message upon successful submission of the form, either by bringing the user to a new page or replacing the form content with the message.

Design specifications

  • 1. Icon

    Each alert type uses the following icons:

    • Danger alerts use an exclamation point icon in a circular border\e101
    • Warning alerts use an exclamation point icon in a triangular border \e107
    • Info alerts use an information icon \e086
    • Success alerts use a checkmark icon \e084
  • 2. Border highlight

    Alerts have border highlights with a thickness of 6px using the following colours:

    • Danger alerts use red #d3080c
    • Warning alerts use red #f90
    • Info alerts use red #269abc
    • Success alerts use red #278400
  • 3. Heading

    Alert heading text colour matches the border highlight colour

  • 4. Description

    Content area of an alert should inherit fonts and styles of the page
