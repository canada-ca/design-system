Link buttons

When to use: Use link buttons for infrequently used options, such as for restarting a wizard or calculator.

Call to action buttons

When to use: Use call to action buttons to start an application, survey, set of interactive questons or another experience when it is top task on that page.

Danger buttons

When to use: Use danger buttons to bring attention to an action that might have consequences, such as deleting a form or leaving an authenticated application.

Usage

Only use call to action buttons for the primary action

Don't

Avoid having 2 call to action buttons on the same page.

Do

Use secondary action buttons if there are various possible next steps with similar frequency or importance.

Generally, don't disable buttons

Do

If a button requires permissions a visitor does not have, or if a button cannot be used conditionally, hide it, don't disable it.

Do

Use form validation to indicate to a visitor that more fields are required before a button's action can be carried out.

Do

Buttons can be disabled once they have been clicked to avoid accidental double-clicks.

Do

Buttons in a toolbar that is consistent across various pages or states can be disabled if not applicable in the current page or situation.

Customisations for buttons

Option Description How to use
Full width Make a button occupy the full width of its column. Add the class btn-block to button.
Grouped buttons Buttons can be grouped side by side, such as for toggle buttons. Surround the grouped buttons with a div with the class btn-group.
Change sizing Buttons can be sized to improve visibility for call to actions or to save space in designs such as a toggle. Apply the following classes to the button:
  • Large: btn-lg
  • Small: btn-sm
  • Extra small: btn-xs
Call a javascript function Call a javascript function to complete the action. ((Would we give options? Leave javascript in its own separate guide?))
