Buttons: Design guidelines

Call to action to let a user complete an action, find a next step or submit a form.

  • Version: 0.1
  • Updated: Dec 24, 2019
Design guidelines Demo and code Rationale

Design variants

Get code

1. Primary button

Use primary buttons for actions someone is likely to take on that page, such as the next or submit buttons in a form.

Get code

2. Secondary button

Use secondary buttons for alternative, less important or less likely to be used options, such as the 'previous' button in a form.

Get code

3. Link button

Use link buttons for infrequently used options, such as for restarting a wizard or calculator.

Get code

4. Call to action button

Use call to action buttons to start an application, survey, set of interactive questons or another experience when it is top task on that page.

Get code

5. Danger button

Use danger buttons to bring attention to an action that might have consequences, such as deleting a form or leaving an authenticated application.

Related patterns

Radio buttons are used in the following patterns:

  • Interactive questions
  • Forms
  • Toggles

Usage

Only use call to action buttons for the primary action

Don't

Avoid having 2 call to action buttons on the same page.

Do

Use secondary action buttons if there are various possible next steps with similar frequency or importance.

Generally, don't disable buttons

Do

If a button requires permissions a visitor does not have, or if a button cannot be used conditionally, hide it, don't disable it.

Do

Use form validation to indicate to a visitor that more fields are required before a button's action can be carried out.

Do

Buttons can be disabled once they have been clicked to avoid accidental double-clicks.

Do

Buttons in a toolbar that is consistent across various pages or states can be disabled if not applicable in the current page or situation.
