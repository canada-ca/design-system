Mandatory on all pages

The report a problem component allows people to provide feedback via a form for any problems they experience with a web page. This component is intended to collect and address any issues for the page on which it resides.

Use this component to enable page-level feedback. Common issues are presented for a person to select using checkboxes. If none of these options are applicable, they may select ‚ÄúOther‚Äù.

On this page

  • When to use
  • What to avoid
  • How to implement
  • Discussion

When to use

The Report a Problem pattern is mandatory on all pages. For more information, go to the Mandatory elements of the design system.

What to avoid

Ensure that the feedback you receive is actionable. If you can, do not implement a form where users cannot give details on the issues they are experiencing.

How to implement

Using CDTS to implement the Canada.ca theme

Do not implement Report a Problem through CDTS. To turn off the page footer in CDTS:

  • use the CDTS version for GCWeb applications, even if it is for your static HTML pages
  • remove the reference for the 'prefooter'

You should then set up the report a problem form in its place.

The button, label, placement and privacy statement are mandatory:

  • the component is presented as a button using the button component for secondary actions, which triggers a feedback form to expand when clicked
  • the label reads: ‚ÄúReport a problem on this page‚Äù
  • placement: positioned directly below the main content area, left-aligned
  • privacy statement: positioned within a dismissible overlay.

The pattern for the contents of the feedback form is recommended. If meaningful data is not being received, a mailto link can be used as an alternative.

Sending the form: Make sure to replace the "#" placeholder in the form action line with wherever your institution can collect the data from the form. Consider sending the form to a generic email inbox if you don't have a server set up to handle this type of form.
