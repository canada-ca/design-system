Checkboxes and radio buttons: Government of Canada design system

Checkboxes and radio buttons with large touch targets to support behavioural patterns of interacting with the form element, and not the label.

On this page

  • Research
  • When to use
  • What to avoid
  • How to implement
  • Discussion

Research

Testing
Explanation of the testing / blog article link

When to use

Checkboxes and radio buttons should be used in interactive questions, such as in application forms for a service, wizards, or checklists.

Use of interactive questions or checklists is highly encouraged on Canada.ca.

What to avoid

Do not use checkboxes or radio buttons if presenting options that are not easily comparable. Consider using navigation patterns such as doormat links.

For a stand alone question with few options and a manageable amount of content per option that does not require separate pages, consider using expand/collapse or tabs.

When to use and how to implement

Large checkboxes

When to use: Use large checkboxes when one or more selections can be made to answer a question. It can also be used for a checklist, with short labels that do not break across lines.

What to avoid: Do not use the large checkbox design when making a checklist with items that break across multiple lines or when there is a single item that can be checked. For these situations, use the small checkboxes.

Which of the following pets do you keep?

<div class="mrgn-tp-lg">

 <ul class="list-unstyled can-ca-form">
  <li class="checkbox">
   <input type="checkbox" id="checkbox_1">
   <label for="checkbox_1">Cat</label>
  </li>
  <li class="checkbox">
   <input type="checkbox" id="checkbox_2">
   <label for="checkbox_2">Dog</label>
  </li>
  <li class="checkbox">
   <input type="checkbox" id="checkbox_3">
   <label for="checkbox_3">Goldfish</label>
  </li>
  <li class="checkbox">
   <input type="checkbox" id="checkbox_4">
   <label for="checkbox_4">Hamster</label>
  </li>
 </ul>

</div>

Small checkboxes

When to use: Use small checkboxes when making a checklist with items that would break across multiple lines or when there is a single item that can be checked.

What to avoid: Do not use the small checkbox design for answers to a question or checklists with labels without line breaks. For these situations, use the large checkboxes.

Single item

 <li class="checkbox checkbox-small">

  <label for="confirmation-checkbox">Home address is the same as mailing address</label>
 </li>
</ul>

Configuration options

Option Description How to configure
Disabled state Disabled inputs will not be clickable. Visual indications include greying out the control and adding a 'stop' cursor icon on hover. To toggle a disabled state:
  • Add the class 'is-disabled' to the list item of the checkbox or radio item
  • Add a "disabled" attribute to the input field
Inline state Inline radio buttons or checkboxes will display side by side on larger screens and wrap to new lines, rather than vertically. Add the "form-label-inline" class to each checkbox or radio button label.
